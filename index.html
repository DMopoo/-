嵌入式八股问题汇总

C 语言 数据结构 操作系统 ARM 通信协议 Linux 驱动 RTOS 调试 — 共 10 章，嵌入式面试高频考点

一、C 语言基础

1.1 关键字与类型

1. static 关键字的作用？
  • 修饰局部变量：延长生命周期（存放在静态区），只初始化一次
  • 修饰全局变量/函数：限制作用域为当前文件（内部链接）
  • 修饰成员变量（C++）：类变量，所有对象共享
2. const 关键字的作用？
  • const int *p：指针指向的内容不可改
  • int * const p：指针本身不可改
  • const int * const p：两者都不可改
  • 修饰函数参数表示函数内不修改该参数
3. volatile 关键字的作用？使用场景？
  • 禁止编译器优化，每次从内存地址读取，不缓存到寄存器
  • 场景：① 硬件寄存器 ② 中断服务程序中修改的全局变量 ③ 多线程共享变量
4. extern 关键字的作用？
声明变量/函数在其他文件中定义，用于跨文件引用
5. sizeof 和 strlen 的区别？
  • sizeof：编译期运算符，计算类型/变量占内存大小（含 \0），可作用于类型
  • strlen：运行时函数，计算字符串长度（不含 \0），只能作用于字符串

1.2 指针

6. 指针数组和数组指针的区别？
  • int *p[5]：指针数组，5 个元素，每个是 int*
  • int (*p)[5]：数组指针，指向含 5 个 int 的数组
7. 函数指针怎么定义？
int (*func_ptr)(int, int);        // 指向 int func(int, int)
int (*func_arr[10])(int, int);    // 函数指针数组
8. 野指针是什么？怎么避免？
  • 野指针：指向已释放内存或未初始化地址的指针
  • 避免：① 初始化 NULL ② free 后置 NULL ③ 不返回局部变量地址
9. void* 指针的理解？
  • 通用指针类型，可指向任意类型，赋值/比较时无需强制转换
  • 解引用前必须转换为具体类型指针

1.3 内存管理

10. C 语言内存分区（内存布局）？
  • 栈：局部变量、函数参数，编译器自动管理
  • 堆：malloc/free 手动管理
  • 全局/静态区（.data / .bss）：全局变量、静态变量
  • 常量区（.rodata）：字符串常量等
  • 代码区（.text）：程序指令
11. malloc / calloc / realloc 的区别？
  • malloc：分配但不初始化
  • calloc：分配并初始化为 0
  • realloc：调整已分配内存大小
12. 内存对齐是怎么回事？为什么需要？
  • 数据存放地址为其大小的整数倍
  • 目的：CPU 一次读取可获取完整数据，提高效率
  • #pragma pack(n) 可指定对齐字节
13. 栈溢出（Stack Overflow）的原因？
递归过深、局部数组过大、函数调用层级过多
14. 大端序和小端序的区别？怎么判断？

大端：高字节存低地址；小端：低字节存低地址

int x = 1;
if (*(char*)&x == 1) printf("小端\n");

1.4 结构体与联合体

15. struct 和 union 的区别？
  • struct：各成员独立存储，大小为各成员大小之和（考虑对齐）
  • union：所有成员共享同一段内存，大小为最大成员的大小
16. 结构体成员对齐规则？
  • 第一个成员偏移量为 0
  • 每个成员偏移量为其大小的整数倍
  • 总大小为最大成员大小的整数倍

1.5 宏与预处理

17. #define 和 typedef 的区别？
  • #define：预处理阶段文本替换，无类型检查
  • typedef：编译阶段类型别名，有类型检查
18. #define 与 inline 函数的区别？
  • #define：预处理器展开，无类型检查，可能有副作用
  • inline：编译器决定是否内联，有类型检查，参数求值一次
19. 宏定义中的 # 和 ## 作用？
  • #：字符串化操作符
  • ##：连接操作符
20. 头文件中的 #ifndef / #define / #endif 作用？
防止头文件被重复包含（头文件保护符）

二、数据结构与算法

2.1 线性结构

1. 数组和链表的区别？
  • 数组：连续内存，随机访问 O(1)，插入/删除可能 O(n)
  • 链表：非连续内存，访问 O(n)，插入/删除 O(1)
2. 栈和队列的区别？常用场景？
  • 栈：LIFO，场景：函数调用、括号匹配、撤销操作
  • 队列：FIFO，场景：任务调度、消息队列、BFS
3. 环形缓冲区（Ring Buffer）如何实现？
  • 用读/写两个指针，模运算实现循环
  • 场景：串口收发、音频处理

2.2 树

4. 二叉树遍历有哪几种？
  • 前序（根→左→右）、中序（左→根→右）、后序（左→右→根）
  • 层序遍历（BFS）
5. 二叉搜索树（BST）的性质？
左子树所有节点值 < 根节点值 < 右子树所有节点值

2.3 查找与排序

6. 常见的排序算法及复杂度？
算法平均时间最坏时间空间稳定？
冒泡O(n²)O(n²)O(1)稳定
选择O(n²)O(n²)O(1)不稳定
插入O(n²)O(n²)O(1)稳定
快速O(nlogn)O(n²)O(logn)不稳定
归并O(nlogn)O(nlogn)O(n)稳定
堆O(nlogn)O(nlogn)O(1)不稳定
7. 二分查找的条件和复杂度？
要求有序数组，O(log n)

2.4 位运算

8. 常见位运算操作（嵌入式高频）？
#define SET_BIT(x, n)    ((x) |= (1u << (n)))      // 置位
#define CLEAR_BIT(x, n)  ((x) &= ~(1u << (n)))     // 清零
#define TOGGLE_BIT(x, n) ((x) ^= (1u << (n)))      // 翻转
#define GET_BIT(x, n)    (((x) >> (n)) & 1u)       // 读取
9. 不用临时变量交换两个数？
a = a ^ b; b = a ^ b; a = a ^ b;
// 或
a = a + b; b = a - b; a = a - b;
10. 判断一个数是否为 2 的幂？
(n > 0) && ((n & (n - 1)) == 0)

三、操作系统

3.1 进程与线程

1. 进程和线程的区别？
  • 进程：资源分配基本单位，独立地址空间，切换开销大
  • 线程：CPU 调度基本单位，共享进程地址空间，切换开销小
2. 进程间通信（IPC）方式有哪些？
  • 管道（pipe）/ 命名管道（FIFO）
  • 信号（signal）
  • 消息队列
  • 共享内存（最快）
  • 信号量（semaphore）
  • Socket
3. 线程同步方式有哪些？
  • 互斥锁（mutex）
  • 读写锁（rwlock）
  • 条件变量（condition variable）
  • 信号量（semaphore）
  • 自旋锁（spinlock）
4. 死锁的四个必要条件？如何预防/避免？
  • 互斥、持有并等待、不可剥夺、循环等待
  • 破坏任一条件：① 超时释放 ② 一次性申请 ③ 优先级继承 ④ 按序加锁 ⑤ 银行家算法
5. 用户态和内核态的区别？切换方式？
  • 用户态：受限指令，不能直接访问硬件
  • 内核态：特权模式，可访问所有资源
  • 切换方式：系统调用、中断、异常
6. 中断的处理流程？
① 保护现场（压栈） → ② 查中断向量表 → ③ 执行 ISR → ④ 恢复现场 → ⑤ 返回
7. 中断上半部和下半部（Top Half / Bottom Half）的区别？Linux 实现方式？
  • Top Half：在中断上下文中执行，要快，通常只做读写寄存器
  • Bottom Half：在工作/线程上下文中执行，处理耗时逻辑
  • Linux 实现：softirq、tasklet、workqueue

3.2 调度

8. 进程/线程状态转换？
  • 就绪 → 运行（被调度）  运行 → 就绪（时间片到）
  • 运行 → 阻塞（等待资源）  阻塞 → 就绪（资源就绪）
9. 常见调度算法？
  • 先来先服务（FCFS）、短作业优先（SJF）
  • 时间片轮转（RR）、多级反馈队列
  • 优先级调度
10. 实时操作系统（RTOS）的调度特点？
  • 可抢占、优先级调度、确定性延迟
  • 常见：FreeRTOS、uC/OS、RT-Thread

3.3 内存管理

11. 虚拟内存和物理内存的关系？MMU 做了什么？
  • 虚拟内存：每个进程看到的独立地址空间
  • MMU：虚拟地址→物理地址转换，提供内存保护
  • 页表：存储映射关系
12. 内存碎片有哪几种？
  • 内部碎片：分配的内存大于实际使用
  • 外部碎片：空闲内存不连续导致无法分配大块

四、ARM 架构与单片机

4.1 ARM 体系结构

1. ARM 的几种工作模式？
User、FIQ、IRQ、Supervisor、Abort、Undefined、System、Monitor（TrustZone）
2. ARM 寄存器组？
  • R0-R12 通用、R13(SP)、R14(LR)、R15(PC)、CPSR/SPSR
  • Banked 寄存器在不同模式下独立
3. ARM 异常/中断的响应流程？
① 保存 CPSR 到 SPSR_mode → ② 设置 CPSR → ③ 保存返回地址到 LR_mode → ④ 跳转到异常向量表 → ⑤ ISR 执行后通过 subs pc, lr 返回
4. 什么是中断向量表？位置在哪？
异常处理入口地址的表格，通常位于 0x00000000 或 0xFFFF0000（高向量）
5. ARM 的 Load/Store 架构是什么意思？
  • 数据操作只能在寄存器中进行，不能直接操作内存
  • 必须先 load 到寄存器，操作后再 store 回内存

4.2 外设与接口

6. GPIO 的输入输出模式有哪些？
推挽输出、开漏输出、浮空输入、上拉输入、下拉输入、模拟输入
7. 中断嵌套怎么处理？
  • 高优先级中断可打断低优先级 ISR
  • NVIC 支持抢占优先级和响应优先级（子优先级）
8. 看门狗（Watchdog）的工作原理？
  • 递减/递增计数器，程序正常运行时会"喂狗"重置计数器
  • 程序跑飞后计数器溢出触发复位
9. DMA（直接存储器访问）的作用和优势？
  • 在外设和内存之间直接搬数据，不需要 CPU 参与
  • 解放 CPU，适合大批量数据传输（如 SPI / I2S / ADC 采样）
10. Bootloader 的作用和启动流程？
  • 作用：初始化硬件、引导应用程序、支持固件升级
  • 流程：① 上电 → ② 初始化时钟/内存 → ③ 检查升级条件 → ④ 跳转 APP

五、通信协议

5.1 串行通信基础

1. 同步和异步通信的区别？
  • 同步：有时钟线配合
  • 异步：依赖波特率约定，起始位+停止位
2. 全双工、半双工、单工的区别？
  • 全双工：同时双向收发（UART / SPI）
  • 半双工：分时双向（I2C / RS-485）
  • 单工：单向传输

5.2 UART

3. UART 的帧格式？
起始位(1) + 数据位(5-9) + 校验位(可选) + 停止位(1 / 1.5 / 2)
4. 波特率是什么？9600 波特率传输 1 字节需要多久？
  • 波特率 = 每秒传输的码元数
  • 1 字节 = 起始位 + 8 数据位 + 停止位 = 10 位
  • 时间 = 10 / 9600 ≈ 1.04ms
5. RS232 和 RS485 的区别？
  • RS232：点对点、电平 ±15V、距离短(~15m)
  • RS485：多设备总线(最多 32 个)、差分信号、距离远(~1200m)、抗干扰

5.3 I2C

6. I2C 总线需要几根线？怎么实现多设备？
  • 2 根：SCL（时钟）+ SDA（数据），开漏输出 + 上拉电阻
  • 通过设备地址区分，7 位或 10 位地址
7. I2C 的起始/停止信号？
  • 起始：SCL 高时，SDA 下降沿
  • 停止：SCL 高时，SDA 上升沿
8. I2C 应答机制？时钟拉伸？
  • 每字节后接收方拉低 SDA 一个时钟表示 ACK
  • 时钟拉伸：从机拉低 SCL 让主机等待

5.4 SPI

9. SPI 总线需要几根线？
  • 4 根：SCLK(时钟) + MOSI(主出从入) + MISO(主入从出) + CS/SS(片选)
  • 每增加一个从设备多一根 CS 线
10. SPI 的四种模式（CPOL / CPHA）？
  • CPOL：时钟极性（空闲时电平高低）
  • CPHA：时钟相位（第几个边沿采样）
  • Mode 0: CPOL=0, CPHA=0 → 上升沿采样（最常用）
  • Mode 3: CPOL=1, CPHA=1
11. I2C 和 SPI 对比？
特性I2CSPI
线数2 根4 根
通信模式半双工全双工
速率100k / 400k / 3.4M几十 MHz
多设备地址机制片选线
多主多从支持有主从之分

5.5 CAN

12. CAN 总线的特点？
  • 差分信号、多主架构、帧 ID 优先级仲裁、硬件错误检测、最高 1Mbps
  • 帧类型：数据帧、远程帧、错误帧、过载帧
13. CAN 总线仲裁机制？
  • ID 越小优先级越高，逐位仲裁，发送显性位(0)的获胜
  • 无损仲裁，不会破坏数据

六、Linux 系统编程

6.1 文件 I/O

1. 标准 IO 和系统 IO 的区别？
  • 标准 IO（fopen / fread）：带用户缓冲区（stdio buffer），跨平台
  • 系统 IO（open / read）：直接系统调用，无用户缓冲，POSIX 标准
2. Linux 下一切皆文件的含义？
普通文件、设备、socket、管道等都用文件描述符操作

6.2 多线程编程

3. pthread_create 创建线程的参数？
int pthread_create(pthread_t *tid, const pthread_attr_t *attr,
                   void *(*start)(void *), void *arg);
4. 互斥锁和条件变量的配合使用？
pthread_mutex_lock(&mutex);
while (!condition)       // 用 while 不用 if，防止虚假唤醒
    pthread_cond_wait(&cond, &mutex);
// 干活...
pthread_mutex_unlock(&mutex);
5. 线程分离（detach）和可联接（joinable）的区别？
  • joinable（默认）：需要 pthread_join 回收，父线程阻塞等待
  • detached：线程结束后自动释放资源，不可 join

6.3 进程管理

6. fork() 的返回值和写时拷贝（COW）？
  • 父进程中返回子进程 PID，子进程中返回 0，失败返回 -1
  • COW：fork 时不立即复制页，父子共享只读，写时再复制
7. 僵尸进程和孤儿进程？
  • 僵尸：子进程结束但父进程未 wait，PCB 残留
  • 孤儿：父进程先退出，子进程被 init (PID=1) 收养
8. exec 系列函数的区别？
  • l(list) vs v(vector)：参数传递方式
  • p：PATH 搜索 vs 绝对路径
  • e：自定义环境变量

6.4 信号

9. 常见信号及含义？

10. signal() 和 sigaction() 的区别？
sigaction 更健壮，不重置处理函数，支持更多选项

6.5 网络编程

11. TCP 三次握手和四次挥手的过程？
  • 握手：SYN → SYN+ACK → ACK
  • 挥手：FIN → ACK → FIN → ACK（TIME_WAIT 2MSL）
12. TCP 和 UDP 的区别？
特性TCPUDP
连接面向连接无连接
可靠性可靠、有序不可靠、无序

开销大小

13. select / poll / epoll 的区别？
APIfd 限制复杂度特点
select1024O(n) 轮询每次需复制 fd_set
poll无限制(链表)O(n) 轮询结构体数组

14. 阻塞 IO 和非阻塞 IO 的区别？
  • 阻塞：无数据时阻塞等待直到数据就绪
  • 非阻塞：立即返回，配合轮询或多路复用使用

七、驱动开发

1. 字符设备驱动的基本框架？
  • register_chrdev() 注册
  • 实现 file_operations 结构体（open / read / write / release）
  • module_init() / module_exit() 入口/出口
2. 设备号（主设备号/次设备号）的作用？
  • 主设备号：对应驱动
  • 次设备号：区分同一驱动下的不同设备实例
3. ioremap() 的作用？
将物理地址映射到内核虚拟地址，使 CPU 可访问外设寄存器
4. copy_from_user / copy_to_user 为什么要用？
内核空间不能直接访问用户空间指针，需要通过这两个函数安全复制
5. 设备树（Device Tree）的作用？
  • 硬件描述文件(.dts)，描述外设的地址、中断、时钟等资源
  • 编译为 .dtb 由 bootloader 传给内核
6. platform 总线驱动模型？
  • 分离 device（资源描述）和 driver（驱动逻辑）
  • 通过 match 函数匹配，probe 后初始化
7. 中断注册 API？上半部 vs 下半部（Linux）？
  • request_irq()：注册中断处理函数（运行在中断上下文，不能 sleep）
  • 下半部：tasklet（中断上下文但开中断）、workqueue（进程上下文可 sleep）、softirq
8. 并发与竞态的保护机制？
  • 自旋锁（spinlock）：忙等，用在不能 sleep 的上下文（如中断）
  • 互斥体（mutex）：阻塞，可 sleep，用于进程上下文
  • 信号量（semaphore）、原子操作（atomic_t）

八、RTOS（以 FreeRTOS 为例）

1. 任务的状态？
Running、Ready、Blocked、Suspended
2. FreeRTOS 的任务调度策略？
  • 基于优先级抢占 + 同优先级时间片轮转
  • 可配置为抢占式或合作式
3. 任务间通信机制？
消息队列（Queue）、信号量（Semaphore）、互斥量（Mutex）、事件组（Event Group）、任务通知（Task Notification）
4. 互斥量优先级继承机制？
防止优先级反转：持有锁的低优先级任务临时提升到等待锁的高优先级
5. 空闲任务的作用？什么情况下会被饿死？
  • 作用：① 释放删除任务的栈空间 ② 执行空闲钩子
  • 更高优先级的任务持续运行会使空闲任务无法执行
6. FreeRTOS 的 Tick 中断？
  • 系统心跳，由硬件定时器产生，触发任务调度和延时管理
  • 默认每 1ms 一次

九、常见陷阱与调试

9.1 常见 Bug

1. 数组越界导致的后果？
可能覆盖相邻变量/栈帧/返回地址，导致崩溃或不可预期行为
2. 缓冲区溢出怎么防？
用 strncpy 代替 strcpy、snprintf 代替 sprintf、限制输入长度
3. 整数溢出的例子？
unsigned int a = UINT_MAX; a++; // 变成 0
4. 未初始化的变量有什么问题？
值是随机的（栈），导致不可预期的行为

9.2 调试手段

5. 常用的调试方式？
  • 硬件：JTAG/SWD + GDB/OpenOCD、逻辑分析仪、示波器
  • 软件：串口日志、断言 assert、core dump、addr2line
6. HardFault 怎么排查？
  • 查看 LR 确定异常发生时的模式
  • 查看 SP 回溯栈帧，找到出问题前的 PC
  • 常见原因：非法内存访问、未对齐访问、除零
7. 内存泄漏怎么检测？
  • 工具：Valgrind（Linux）、mtrace
  • 嵌入式：自行包装 malloc/free 统计，或看 heap 水位

1. 项目中最难的 Bug？怎么解决的？
（准备一个具体案例，按：现象→定位→根因→修复→验证 的线索描述）
2. 如果串口收数据有丢包，怎么排查？
  • 检查波特率匹配、中断优先级、是否关中断时间过长
  • 用逻辑分析仪抓波形
  • 增大 FIFO 深度或使用 DMA
3. 如何降低嵌入式系统的功耗？
  • 睡眠模式/深度睡眠、降频、关闭不用的外设时钟
  • 用中断代替轮询、选低功耗器件
4. 从上电到 main 函数，经历了什么？
  • ① 硬件复位 → ② 加载 SP 和 PC (Reset_Handler) → ③ 系统初始化(时钟/中断向量表)
  • ④ 初始化 .data 段（从 Flash 复制到 RAM）、清 .bss 段 → ⑤ 调用 main
5. 如果 Flash 空间不够了，怎么优化？
  • 编译优化 -Os、删除无用代码和库函数、压缩字符串/数据、考虑外部 Flash
